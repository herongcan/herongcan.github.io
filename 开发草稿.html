开发草稿1

本文为开发过程中的问题速记，已从静态页迁移到源码仓，后续会持续整理为结构化笔记。

导入 Spring 源码包报错

  • 问题：JDK 上下文不匹配导致构建报错。
  • 记录：检查 JDK 与项目配置一致性，排查本地环境差异。

MySQL 服务清理

  • 控制面板卸载后，补充清理注册表服务项。
  • 命令行可使用 sc delete <serviceName> 删除残留服务。

Vue 项目依赖缺失

  • Cannot find module 类报错可先清理依赖后重装：npm install。

SQL 脚本与提交规范

  • 提交代码建议携带建表、变更、查询示例脚本，便于联调与回溯。

Kettle 使用记录

  • 区分 ktr（转换）与 kjb（作业）。
  • 支持命令行和批处理方式执行。

UNION / UNION ALL

  • 关注列对齐与排序字段一致性。
  • 多数据源汇总时可用 UNION ALL 纵向拼接后统一排序。

其他速记

  • HTML 修改不生效：检查应用更新策略与缓存。
  • 页面空白区点击事件：注意遮罩层与事件冒泡。
  • 元素层级：使用 z-index 管理显示优先级。
